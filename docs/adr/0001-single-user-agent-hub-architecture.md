# 单用户 Agent–Hub 架构：本地代理在请求路径上，中心服务持权威状态

switchAPI 服务单个用户的多台设备。每台设备运行常驻守护进程 Agent，Claude Code/Codex 的请求经 localhost 上的 Agent 直连供应商（不绕经服务器）；常开服务器上的 Hub 持有唯一权威数据（供应商库、全局切换状态、用量汇总），向 Agent 推送配置并汇聚其上报的用量。桌面壳与 Web 控制台共用同一套前端，均连接 Hub。

## Considered Options

- **中心网关（claude-code-hub 式，AI 流量绕经服务器）**：统计与多端同步实现最简单，但所有推理流量多一跳延迟，且服务器宕机会阻断所有设备的 AI 请求。被拒。
- **多机 agent 改写各工具配置文件（cc-switch 式）**：请求零绕行，但用量只能靠解析各机本地日志按切换时间线拼接（有误差），且切换后需重启终端。被拒。

## Consequences

- 组件三件套（Hub、Agent、双壳前端）是全项目最大的工程成本来源，这是有意识接受的代价。
- Agent 缓存最后已知配置与备选序列：Hub 宕机不阻断 AI 请求，仅失去切换、统计上报与多端同步。
