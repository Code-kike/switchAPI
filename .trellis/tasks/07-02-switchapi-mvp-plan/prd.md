# switchAPI — 产品需求文档（PRD）

> 本文档是 2026-07-02 grilling 会话的正式产出，是后续开发的规划基准。
> 领域术语见根目录 `CONTEXT.md`；定位级决策见 `docs/adr/0001`–`0005`。

## 1. 背景与定位

为**单个用户的多台设备**提供 Claude Code / Codex 的供应商（中转站/官方 API）一键切换、
数据备份导出、精确用量与费用统计，桌面应用与远程 Web 双端实时一致。

对标与差异：
- 相对 cc-switch：砍掉多工具支持与配置管家类功能（MCP/Prompts/Skills 等），
  换取多设备一致性、代理路径精确统计、Web 远程访问。
- 相对 claude-code-hub：去掉多用户/发 key/限流/负载均衡，AI 流量不绕经中心服务器。

## 2. 系统架构（已定，ADR-0001）

```
设备1: CC/Codex ─► Agent(127.0.0.1 守护进程) ─► 供应商A/B/C（直连，不绕服务器）
设备2: CC/Codex ─► Agent(127.0.0.1 守护进程) ─► 供应商A/B/C
                      ▲ 配置推送 / 用量上报（断连缓冲补报）
常开服务器: Hub（权威 SQLite + 统计汇聚 + 配置分发 + Web 托管）
                      ▲
        桌面壳(Tauri) 与 浏览器 共用同一套 React SPA
```

组件职责：
- **Agent**（Go，单二进制，systemd/launchd/Windows 服务）：同格式直通转发（Anthropic→Anthropic
  兼容、OpenAI→OpenAI 兼容，仅换 auth 头 + 可选模型名重定向，SSE 透传）；用量采集上报；
  配置缓存与断连临时降级；首次安装写一次 CC/Codex 配置指向本机；一次性配对码接入 Hub。
- **Hub**（Go，单二进制/Docker，前端 embed）：唯一权威数据；切换裁决（含故障切换）；
  配置实时推送；用量汇聚与三层计价；备份/导出/导入；鉴权与设备 token 管理。
- **桌面壳**（Tauri 2）：装载同一 SPA + 托盘快切 + 开机自启 + Agent 安装/启停托管 +
  断网应急视图（直连本机 Agent）。

## 3. 已确认决策总表

| # | 决策点 | 结论 |
|---|--------|------|
| 1 | 用户模型 | 单用户个人工具，明确不做多用户/发 key |
| 2 | 拓扑 | 多台设备 + 常开服务器（Hub 部署点，内网/Tailscale） |
| 3 | 核心架构 | 本地代理在请求路径 + 中心汇聚（拒绝中心网关绕行、拒绝改配置文件方案） |
| 4 | Agent 形态 | 独立守护进程；桌面壳引导安装与托管；纯 CLI 也可安装 |
| 5 | 双端 UI | 一套 React SPA，Web 由 Hub 托管，桌面为 Tauri 壳 |
| 6 | 协议边界 | 同格式直通，不做跨协议转换（ADR-0002） |
| 7 | 切换语义 | 全局唯一状态，per-App；一次切换全设备生效；数据模型预留设备维度 |
| 8 | 用量语义 | 本系统代理路径精确统计（MVP）+ 中转站余额适配器（二期） |
| 9 | 计价 | 三层叠加：LiteLLM 基准价（自动同步+内置快照）→ 供应商折扣系数 → 模型覆盖价；缓存 token 分项 |
| 10 | 备份导出 | 本地快照轮转（每日+变更后）；导出 JSON 含密钥强制口令加密；明细永久保留+CSV；cc-switch 一键导入 |
| 11 | 安全 | 内网优先不内置 TLS（ADR-0005）；密码+Session；配对码换可吊销设备 token；密钥 AES-GCM 落库 |
| 12 | 技术栈 | Go + React/TS/Vite/Tailwind/shadcn + Tauri 2 + SQLite（ADR-0003/0004） |
| 13 | 范围 | 见第 4 节四象限 |
| 14 | MVP 附加 | 预设模板、托盘快切、故障自动切换、端点测速全部进 MVP（用户明确选择，后两项超出原推荐） |
| 15 | 故障切换语义 | Agent 上报 → Hub 裁决 → 沿备选序列全局切换 + 通知；断连时本地临时降级，重连对齐 |

## 4. 功能范围四象限

### MVP（一期交付）
1. 供应商管理：CRUD、预设模板、协议类型、折扣系数、模型覆盖价、备选序列
2. 一键切换：Web / 桌面 / 托盘三入口，全局热生效（下一请求走新供应商，无需重启终端）
3. 用量统计：请求明细（token 分项/耗时/状态/归属）+ 按日/供应商/模型/App/设备聚合 + 趋势图 + 费用仪表盘
4. 故障自动切换：健康上报、Hub 裁决、备选序列、Web + 桌面通知
5. 端点测速：手动触发的延迟/可用性探测
6. 备份导出：快照轮转、加密 JSON 导出/导入、CSV 用量导出、cc-switch 迁移导入
7. 双端：Web 控制台 + Tauri 桌面壳（含 Agent 托管、断网应急视图）
8. 配对与安全：管理员密码、设备配对/吊销、密钥加密落库

### 二期
余额查询适配器（one-api/new-api 优先 + 自定义请求模板）、WebDAV 备份推送、
公网直挂安全全套（内置 ACME/TOTP/登录限速）、设备级切换覆盖。

### 停车场（不排期、不写死、不预留设计）
MCP/Prompts/Skills 管理、会话管理器、限流/负载均衡/Deep Link、跨格式协议转换。

### 明确不做
其他工具支持（Claude Desktop/Gemini CLI/OpenCode/OpenClaw/Hermes）、多用户体系。

## 5. MVP 验收标准

- [ ] 两台设备 + 一台服务器完成部署；CC 与 Codex 各配置 ≥2 个供应商
- [ ] 任一入口切换后，所有设备的下一个请求即走新供应商，终端无需重启
- [ ] 人为打断当前上游后，系统沿备选序列自动切换并在双端收到通知
- [ ] 费用统计与中转站后台账单在折扣系数正确配置下误差可解释
- [ ] 关停 Hub：各设备 CC/Codex 照常可用（缓存配置）；重连后用量补报无丢失
- [ ] 备份快照可完整恢复；加密导出文件可在全新 Hub 上导入还原
- [ ] cc-switch 存量供应商一键导入：可映射项全部导入成功，不可映射项（协议转换类/OAuth 托管类/
  回环地址/空 key，见 research/05 drop-list）逐条报告跳过原因
- [ ] Agent 内存常驻占用 < 50MB，流式响应无可感知卡顿

## 6. 待研究项（进入实现前的 research 清单）

> **状态：8/8 已完成**（2026-07-03/04），成果在本任务 `research/01`–`08`.md，结论已回写 design.md。
> 各篇含来源引用与本机实证；#6 附可运行原型（06-sse-prototype/）并已对真实中转站实测通过。

1. Claude Code 指向本地代理的配置写法（`ANTHROPIC_BASE_URL`/`ANTHROPIC_AUTH_TOKEN`、
   settings.json env 块）及配置变更热生效行为的实测验证
2. Codex 的 `config.toml` model_providers + `auth.json` 写法；Responses API 直通与 usage 字段解析
3. Anthropic 流式 usage（message_start/message_delta）与 OpenAI Responses usage 的采集点
4. LiteLLM 价格表字段映射（含 cache_creation/cache_read 单价）、同步频率与快照打包
5. cc-switch 现行数据格式（`~/.cc-switch/` config.json 或 SQLite schema）→ 导入映射表
6. Go SSE 透传实现要点（flush 时机、读写超时、连接复用、大响应内存）
7. Tauri 2 sidecar/自更新与 Agent 二进制分发方案（含 Windows 服务注册）
8. 健康判定默认阈值（连续失败次数、超时窗口、恢复探测）与误判防抖

## 7. 里程碑建议

- **M1 数据与转发内核**：Hub 数据模型 + Agent 直通转发 + 手动切换（CLI 级验证）
- **M2 统计与计价**：用量采集/上报/汇聚 + 三层计价 + 仪表盘
- **M3 双端体验**：Web 控制台完整功能 + Tauri 壳 + 托盘 + 配对流程
- **M4 可靠性**：故障自动切换 + 测速 + 备份导出/导入 + cc-switch 迁移
