# switchAPI

单用户、多设备场景下的 AI 编码工具（Claude Code / Codex）供应商切换与用量中枢。为一个人服务，不做多租户。

## Language

### 核心概念

**供应商（Provider）**:
可被切换的上游推理 API 服务，持有 base_url 与 API Key。可以是官方 API，也可以是第三方中转站。
_Avoid_: 渠道、站点、中转代理（易与"本地代理"混淆）

**应用（App）**:
被 switchAPI 接管供应商配置的 AI 编码工具。目前仅指 Claude Code 与 Codex 两种。
_Avoid_: 工具、客户端

**切换（Switch）**:
更改某个 App 当前生效供应商的操作。作用域为全局：一次切换对所有设备同步生效。每个 App 各自独立持有当前供应商。
_Avoid_: 换源、切站

**设备（Device）**:
安装了 Agent 的一台机器。设备不持有独立的供应商选择，只执行全局状态（Agent 与 Hub 断连期间的临时降级除外）。
_Avoid_: 节点、终端、客户端机器

### 系统角色

**中心服务（Hub）**:
部署在常开服务器上的权威数据与控制中枢：存储供应商库、汇聚用量、分发配置、承载 Web 访问。
_Avoid_: 服务端、后端、网关

**本地代理（Agent）**:
运行在每台开发机上的常驻进程，App 的推理请求经它直连当前供应商；同时负责向 Hub 上报用量、接收配置推送。本项目文档中 Agent 一律指此进程，不指 AI agent。
_Avoid_: 客户端、daemon、节点

**桌面壳（Desktop）**:
桌面应用形态的控制界面。
_Avoid_: 桌面端、GUI

**Web 控制台（Web UI）**:
通过浏览器访问 Hub 的控制界面，与桌面壳功能对等。
_Avoid_: 网页版、前台

### 统计与计价

**用量记录（Usage Record）**:
一次 App 请求经 Agent 转发所产生的计量事实：token 分项（输入/输出/缓存写/缓存读）、耗时、状态，归属到供应商、模型、App 与设备。永不包含消息内容。
_Avoid_: 请求日志、账单

**费用（Cost）**:
对用量记录按三层规则（基准价 → 供应商折扣系数 → 模型覆盖价）折算出的金额估计。
_Avoid_: 价格、消费

**折扣系数（Cost Coefficient）**:
供应商级乘数，表示该供应商相对官方基准价的实际计费比例。
_Avoid_: 倍率、汇率

**余额（Balance）**:
中转站账户侧的剩余额度，经余额适配器从站点 API 查得；是外部事实，不由本系统计量推导。
_Avoid_: 额度、配额

### 接入与可靠性

**配对（Pairing）**:
设备首次接入 Hub 的认证握手：一次性配对码换取可吊销的长期设备 token。
_Avoid_: 注册、登录

**预设模板（Preset）**:
内置的供应商配置模板（base_url、协议、计价系数预填），添加供应商时仅需补全 API Key。
_Avoid_: 预设站点、内置源

**备选序列（Fallback Order）**:
每个 App 维护的供应商优先顺序，是故障切换的依据。
_Avoid_: 备胎列表、优先级组

**故障切换（Failover）**:
Hub 依据 Agent 上报的上游健康状况裁决执行的自动全局切换，沿备选序列选择新供应商。
_Avoid_: 自动降级、容灾

**冷却（Cooldown）**:
供应商被故障切换降级后的一段隔离期，期间不参与备选选取；随连续降级次数指数延长。
_Avoid_: 惩罚期、封禁

**恢复探测（Probe）**:
Hub 指令单个在线 Agent 对被降级供应商发起的最小真实请求，用于判定其是否恢复；恢复后仅通知，不自动切回。
_Avoid_: 健康检查（易与 Agent 的被动健康判定混淆）、心跳

**临时降级（Local Fallback）**:
Agent 与 Hub 断连期间，按本地缓存的备选序列自主偏离全局状态的临时路由；重连后向全局状态对齐。
_Avoid_: 离线模式
